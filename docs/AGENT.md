## 定義
(docsのDefaultとLv1～Lv2)の中身の事を設計書といいます。
(docsのAGENT.mdとDefaultとLv1～Lv3)の事をコードルールなどのニュアンスでいうことがあります。
(docsのLv4)のことを実装状況と呼びます。

## 実装方針
IMP-A.Lv4に実装状況が記載済です。
IMP-B.ページの実装がOKとなった段階で必ず確認を取ります。
実装がOKな場合、以後のほかの実装エージェントと齟齬が発生しない為にLv2～Lv4のDocsを更新します。
ここで競合が発生するのが致命的なので、工夫整合性を保つようにしてください。

## コード生成ルール
CODE-001:ソースコードをそのまま読むことはCSS関連以外基本的にないため、エージェントが分かればOKで、冗長なコメントなどは排除してよいです。
ただし、コードを圧縮するために一見わかりづらい処理を実装してしまうのはやめてください。
CODE-002:基本的に設計書とmocksをベースに1機能ずつ生成します。
生成する際はその他のものを参照しないでも開発が進められるように生成してください。
CODE-003:IMP-Aの実装タスクとIMP-Bの設計書更新タスクは必ずセットで実施してください。
